◈ git, Github 개요 ◈
repository : 깃으로 파일이 저장관리되는 공간
local repository : 팀원 각각의 개인 파일 저장 공간
remote repository : 세팀원이 공유하는 원격 저장 공간
git : 소스코드 및 프로젝트 파일의 버전관리, 이전에 저장된 시점으로 복원가능
GitHub : 팀원들과 프로젝트 파일 공유, 관리 및 다양한 오픈소스 프로젝트 공개
Clone : 최초로 깃허브에서 프로젝트를 불러오는 것

같은파일 수정 -> 충돌 -> commit을 하여 충돌이력을 수정

◈ 팀원들과 사용시 주의점 ◈
- 처음에는 충돌이 일어날 수 있으니 각자 작업하는 클래스 분리
- pull을 받기전에는 내가 작업중이던 파일을 commit 하기 ★★★
  (commit안했는데 원격저장소에서 pull 하려하면 충돌이남)
- push 전에는 내가 github에서 모든 pull 내역을 받았는지 확인 ★★★
 
◈ 내 소스코드를 깃허브에 어떻게 올리나 ◈
1. Terminal 로 간다.
2. git init : git을 쓸 준비. 즉, 내가 초기화를 해주겠다.(이것은 맨 처음 프로젝트를 올릴때)
3. git add . : 점은 전부라는 뜻, 어떤 파일을 깃에 올릴지는 git add, 뒤에 점이 있으니 전부올리겠다
( git add index.html -> index.html 만 올리고 싶다 )
4. git status : git add . 을 해서 내가 뭘추가했는지 알려주는것 (상태알려줌, 필수아님)
5. git commit -m "first commit" : git commit은 나의 히스토리를 만드는 것
6. git remote add origin git@github.com:(나의깃허브사이트) : 내 소스와 깃허브와 연결시켜줌
7. git remote -v : 깃허브와 연결됐는지 확인 (선택사항)
8. git push origin master : master branch로 코드를 보낸다
-> 이렇게 코드를 깃허브에 보낸후 내가 다시 코드를 업데이트 한 후 그 내용을 다시 보내고싶을때
1. git add . (추가 작업에는 git init할 필요없음)
2. git commit -m "second commit"
3. git push origin master

◈ 깃허브에서 프로젝트 다운받는법 ◈
1. code에서 링크복사
2. cmd에 들어가서 cd를 통해 원하는 저장 장소지정
3. git clone 복사한링크(깃헙주소) 폴더이름(여기서 freshman이라 지정함)
(참고로 폴더이름 안써주면 자동으로 프로젝트 이름의 폴더로 들어감)
4. cd 폴더이름 -> 해당 폴더로 들어감
5. code . 을 치면 해당 코드가 vscode에서 실행됨

◈ 코드가져와서 작업한 후 깃허브에 올리는법 ◈
1. git add.
2. git comit -m "freshman first commit"
(이러고 나서 git push origin master하면 ㅈ됨. master는 프로젝트 최종본임)
3. git checkout -b freshman(내 폴더이름) : freshman을 위한 공간(브랜치)을 만들어줌
4. git push origin freshman(내 폴더이름)
5. 깃허브사이트에가서 Compare & pull request 를 누른후 멘트작성하고 Create pull request 누르기 
( *** 이거 안해서 충돌났나? *** )

◈ master에 변화가 생기면 무조건 pull을 해줘야함(동기화) ◈
1. git add .
2. git commit -m "second commit" 
(이거 하고나서 ★push 안해줌★ 대신, pull 먼저 해줘야함 : 충돌방지)
3. git pull origin master
4. git push origin master

(참고로 Terminal에서 ctrl+k 를 누르면 Terminal 내용이 싹 없어짐)
