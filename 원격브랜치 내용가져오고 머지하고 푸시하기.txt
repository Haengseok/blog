git checkout -b cjh origin/server_jack

git checkout -b 이 명령어는  cjh해당하는 부분이 본인이 원격저장소를 로컬로가져와서 부를 이름입니다. 이 명령어를 쓰면 원격저장소를 가져오기만 하는게 아니라 연결도 되어서 거기로 푸시가 가능해집니다.

git checkout시에 파일버전들이 차이가 나면 한번 푸시하고 진행하라고하는 영어문구가나오느데 그대로 하시면 됩니다. 보통은 그냥 그 주의 문구대로 수행하고 그 명령어를 수행하시면 제대로 됩니다. 영어 다 읽어주셔야합니다.

예를 들면 
git branch해서 나온 결과가

*master
cjh
sjh 

이면 로컬에 총 3개의 브랜치가 있다는 뜻입니다. git checkout cjh하면

master
*cjh
sjh

이런식으로 넘어갑니다. git branch server_jack이라고 명령어를 치시면 원격저장소의 브랜치와 이름이 똑같지만 아예 다른 별개의 로컬 브랜치가 생기는 것 뿐입니다.
원격저장소의 branch는 원격저장소의 별칭인 origin/ 를 붙여서 origin/server_jack와 같이 지칭합니다.

행석님의 경우 아예 원격저장소에 본인의 브랜치를 개설하지 않았기 때문에
아마도 git checkout -b local_master origin/master를 하면 
로컬로 local_master 라는 브랜치를 만들어서 최신 프론트엔트 작업물을 가져오실 수 있을 겁니다.
이때 주의하실 것은 저 명령어는 원격저장소의 master와 연결되어있기때문에 그 브랜치에서 푸시하면 원격저장소의 master에 푸시됩니다.

git checkout -b local_server_jack origin/server_jack로 두분 원격브랜치중에 하나 가져와서 두 분 작업물 merge후에 local_master로 머지하시고 푸시하셔야합니다. 머지는 현재 브랜치에 타켓 브랜치를 합병하는 것입니다.

https://trustyoo86.github.io/git/2017/11/28/git-remote-branch-create.html
를 보시고 원격저장소에 본인의 브랜치를 생성하실 수 있을겁니다. push를 하면 되는거 같은데 한번 확인해보세요

종혁님의 경우도 git checkout -b local_master origin/master 로 받으시면 될 겁니다.

두분 다 git branch를 통해 

*master
local_master
sjh

이런식으로 나오는지 확인하고 local_master를 본인이 가지고 있는 브랜치에 merge해주세요

본인의 작업용 로컬 브랜치인지 확인하고 git merge local_master하면 될 겁니다.

충돌 날 수도 있는 데 그건 오류 문구 자세히 읽고 해주시면 됩니다.
